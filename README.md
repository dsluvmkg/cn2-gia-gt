## CN2 GIA和CN2 GT是什么？先搞懂这两个名字

CN2是中国电信推出的下一代骨干承载网络（ChinaNet Next Carrying Network），专门针对跨境传输中高峰丢包、延迟抖动这些老问题设计的。它和传统的163骨干网（AS4134）并行运行，走的是独立的59.43高速节点（AS4809）。

但CN2内部还分两个档次：

**CN2 GT（Global Transit）**：中端产品。数据在国际出口段会经过2到4个59.43的CN2节点，但一旦进入国内省级网络，就切换回202.97的163骨干网节点。可以理解为"出海段走高速，进城段堵主路"。

**CN2 GIA（Global Internet Access）**：顶端产品。从海外数据中心到国内用户，全程走59.43的CN2高速节点，不经过163共享骨干网，有独立的优先级通道。延迟更稳，丢包率更低，就算晚高峰也不掉链子。

简单说：**CN2 GT是半程CN2，CN2 GIA是全程CN2**。这一个"半"字，在平峰时段区别不明显，晚高峰一拉差距就出来了。

CN2 GIA不走163共享出口，有独立的优先级通道。晚高峰实测丢包率接近0%，延迟抖动也小很多。如果你的主要使用场景在傍晚到深夜，这个差异非常直接。

**一句话总结**：白天用什么差距不大，晚上用才看出线路真实水平。

## 怎么判断你的VPS走的是哪条线？

买完VPS，不少人想确认一下到底走的是CN2 GT还是CN2 GIA。方法很简单，用路由追踪工具（traceroute或mtr）测试到国内IP的路径，然后看跳点。

判断规则：

1. **全程是202.97开头的节点** → 普通163线路，没有CN2
2. **国际段有59.43节点，进国内变回202.97** → CN2 GT（半程CN2）
3. **全程大量59.43节点，省级段也是59.43** → CN2 GIA（全程CN2）

一般来说，CN2 GIA的路径中59.43节点出现4个以上，而CN2 GT只有2到3个，且省级段会切回202.97。

Linux/macOS终端可以用 `mtr -rn [目标IP]` 持续测试，结果会比单次traceroute更可靠，还能看到每一跳的丢包率。
